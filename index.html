Описание команд Git

Git Flow

Git Flow — это набор соглашений о том, как использовать Git для управления разработкой программного обеспечения. Этот метод предполагает использование нескольких веток, таких как:

  • main: основная ветка, содержащая стабильный код.
  • develop: ветка для интеграции новых функций.
  • feature/: ветки для разработки новых функций.
  • release/: ветки для подготовки к релизу.
  • hotfix/: ветки для быстрого исправления ошибок.

Git Workflow — это процесс и структура, для команд чтобы управлять проектом, используя ветки (Я ПОНЯЛ), коммиты и слияния для совместной работы и поддержания чистоты кода.

Git Rebase

Git Rebase — это команда, которая позволяет перенести изменения из одной ветки в другую в отличии от git merge, который создает новый коммит слияния, git rebase переписывает историю, перемещая коммиты на верхушку другой ветки.

Пример использования: git checkout feature-branch
git rebase main

Это позволит интегрировать изменения из ветки main в вашу ветку feature-branch, сохраняя линейную историю.

Git Fetch

Git Fetch — это команда, которая загружает изменения из удаленного репозитория, не сливая их с вашей текущей веткой. Это очень круто для получения информации о новых коммитах и ветках без изменения вашей локальной области.

Пример использования: git fetch origin

После выполнения этой команды вы можете просмотреть изменения, используя git log или переключиться на новую ветку.
